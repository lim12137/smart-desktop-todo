# 待办事项提醒系统 - C/S 架构部署指南

## 概述

本项目已从单机桌面应用改造为客户端-服务器（C/S）架构，支持多用户协作：

- **服务端**: Flask REST API + SQLite 数据库
- **客户端**: tkinter 桌面应用，通过 HTTP 与服务器通信
- **部署环境**: 本地局域网（LAN）

## 系统要求

### 服务端
- Python 3.7+
- 局域网内的固定IP地址
- 端口 5000 可用

### 客户端
- Python 3.7+
- 可访问服务端网络
- Windows 系统（支持系统托盘和Toast通知）

## 安装步骤

### 1. 安装依赖

```bash
pip install flask flask-cors pyjwt requests
```

可选（客户端Windows系统增强功能）：
```bash
pip install win10toast pystray pillow
```

### 2. 初始化数据库

首次启动服务端时，会自动创建数据库并生成默认管理员账户：
- **用户名**: admin
- **密码**: admin123
- **IP**: 127.0.0.1

⚠️ **重要**: 首次登录后请立即修改管理员密码！

## 启动指南

### 服务端启动

在项目根目录运行：

```bash
python start_server.py
```

或直接运行：

```bash
cd server
python app.py
```

服务端将监听 `0.0.0.0:5000`，允许局域网内所有设备访问。

### 客户端启动

在项目根目录运行：

```bash
python start_client.py
```

或直接运行：

```bash
cd client
python main.py
```

## 使用流程

### 1. 管理员登录

1. 启动客户端，出现登录对话框
2. 输入服务器地址（如：`http://192.168.1.100:5000`）
3. 输入管理员凭据（admin / admin123）
4. 点击"测试连接"验证网络
5. 点击"登录"

### 2. 创建成员账户

管理员登录后，可通过用户管理功能创建成员：

```python
# 示例：创建成员
用户名: member1
显示名称: 张三
IP地址: 192.168.1.101
角色: member
```

### 3. 分配任务

管理员可以：
- 创建任务并分配给指定成员
- 设置任务优先级、截止日期
- 跟踪所有成员的任务进度

### 4. 成员使用

成员登录后可以：
- 查看分配给自己的任务
- 创建个人任务
- 更新任务进度
- 标记任务完成

## API 端点

### 认证
- `POST /api/auth/login` - 用户登录
- `GET /api/auth/me` - 获取当前用户信息

### 任务管理
- `GET /api/todos` - 获取任务列表
- `GET /api/todos/<id>` - 获取单个任务
- `POST /api/todos` - 创建任务
- `PUT /api/todos/<id>` - 更新任务
- `DELETE /api/todos/<id>` - 删除任务
- `POST /api/todos/batch` - 批量更新状态

### 提醒
- `GET /api/todos/reminders/pending` - 获取待提醒任务
- `POST /api/todos/reminders/<id>/<type>/acknowledge` - 确认提醒

### 用户管理（仅管理员）
- `GET /api/users` - 获取用户列表
- `POST /api/users` - 创建用户
- `PUT /api/users/<id>` - 更新用户
- `DELETE /api/users/<id>` - 删除用户

## 提醒机制

### 服务端提醒服务
- 后台线程每1小时检查一次待提醒任务
- 自动标记提醒状态（7天、3天、逾期）
- 避免重复提醒

### 客户端提醒轮询
- 每5分钟向服务器请求待提醒任务
- 显示系统托盘通知
- 播放提示音（Windows）

## 性能优化

### 客户端优化
- **本地缓存**: SQLite 缓存常用数据，减少网络请求
- **异步加载**: 使用线程池异步获取数据，避免界面卡顿
- **批量操作**: 合并多个更新请求为单次API调用

### 服务端优化
- **索引优化**: 为常用查询字段添加索引
- **连接管理**: 使用上下文管理器确保数据库连接正确释放

## 故障排查

### 客户端无法连接服务器
1. 检查服务器是否正在运行
2. 验证服务器地址是否正确（`http://IP:5000`）
3. 确认防火墙允许5000端口通信
4. 尝试用浏览器访问 `http://服务器IP:5000/api/health`

### 登录失败
1. 确认用户名和密码正确
2. 检查IP地址是否与管理员设置的一致
3. 查看服务器日志获取详细错误信息

### 提醒不工作
1. 服务端：确认提醒服务已启动（查看启动日志）
2. 客户端：确认提醒轮询器正在运行
3. 检查任务是否设置了截止日期
